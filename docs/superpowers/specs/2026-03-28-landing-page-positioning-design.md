# Landing Page Positioning Design

> Replace the current "AI Fanfiction Generator" framing with a homepage that reflects the actual product we are building.

## Problem

The current homepage in [page.tsx](c:\Users\Lenovo\Desktop\fanfic-generator\src\app\page.tsx) still presents the product as:

- `AI Fanfiction Generator`
- "Unlimited personalised stories in your favourite universes"

That no longer matches the product.

The app now includes:

- Codex
- Updates
- Context Console
- continuity-aware generation
- adaptation workflows
- saved adaptation artifacts

The homepage should now explain the system, not just the old entry point.

## Positioning Goal

Public-facing goal:

`an AI writing studio with living project memory`

Internal strategic framing:

`the OS for writing`

Important note:

The landing page should not lead with "OS" language alone. That phrase is strategically useful, but not self-explanatory enough for the first screen.

The page should explain the value in concrete terms.

## Core Messaging

### Hero

Headline directions:

- `Write with AI that actually remembers your project`
- `From first draft to finished outputs, all in one writing studio`
- `A writing platform built for long-form work, living memory, and multi-format outputs`

Subheadline direction:

- Draft, revise, track story truth, adapt chapters into new formats, and keep your project coherent as it grows.

Primary CTA:

- `Start Writing`

Secondary CTA:

- `See How It Works`

### Core Value Pillars

The page should frame the product around four pillars:

1. `Write`
2. `Remember`
3. `Adapt`
4. `Review`

Those are simpler and more concrete than listing raw features.

## Recommended Landing Page Structure

### 1. Hero

Purpose:

- explain the product quickly
- reposition away from "fanfic generator"
- keep the promise concrete

### 2. Value Pillars

Short cards:

- `Write`
  - draft, rewrite, expand, brainstorm, describe
- `Remember`
  - Codex, context, chapter-aware truth, project memory
- `Adapt`
  - summaries, recaps, teasers, screenplay beats, more later
- `Review`
  - updates, continuity, context controls, story consistency

### 3. Product System Section

This section should explain how the system works:

- `Entries`
- `Updates`
- `Context`
- `Artifacts`

Simple explanation:

- build project truth
- review changes
- control what AI uses
- turn one project into many outputs

### 4. Use Cases Section

This should expand beyond fanfic while staying disciplined.

Recommended initial set:

- fanfic
- novels
- short stories
- serialized fiction
- newsletters
- screenplays

Do not try to market every imaginable mode at once.

### 5. Example Workflow Section

Show concrete workflows:

- `Draft chapter -> update Codex -> continue story`
- `Draft chapter -> generate recap -> generate teaser`
- `Track character progression -> review continuity -> revise scene`

This is important because the product is now more system-like and benefits from showing flows, not just static features.

### 6. Vision Section

Short forward-looking section:

- long-form writing
- shared project memory
- multi-format adaptation
- continuity and review
- future mode packs

This is where the larger "writing OS" vision can peek through without making the whole page sound abstract.

### 7. Final CTA

Strong CTA into the app:

- `Start Your First Project`

Optional supporting line:

- Start with fiction today. Expand into more modes as your workflow grows.

## Visual Direction

The page should feel:

- premium
- editorial
- intentional
- not "generic AI SaaS"

Recommended direction:

- dark, ink-like atmosphere is okay if done with care
- strong typography hierarchy
- richer depth and layered panels
- more visual storytelling, less dashboard blandness
- avoid clutter and feature soup

It should feel like a creative tool for serious work, not a toy generator.

## Content Rules

The page should:

- lead with user value, not model names
- explain the loop clearly
- show multiple use cases without losing focus
- avoid overclaiming "everything for everyone"

The page should not:

- sound like only a fanfic tool
- sound like only a content marketing tool
- sound like a generic AI writer clone

## Proposed Near-Term Naming Direction

We do not need a final brand rename before redesigning the landing page.

For now, the safest move is:

- keep the existing product/repo identity in code where needed
- update the public language to a broader, better framing

Examples of acceptable interim public labels:

- `AI Writing Studio`
- `Writing Studio with Living Memory`
- `Creative Writing Workspace`

The brand naming conversation can happen in parallel, but it should not block the landing page upgrade.
